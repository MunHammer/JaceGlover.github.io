# AceBlocks Security Policy

AceBlocks is committed to keeping our platform and code safe for everyone. This document explains how to report security issues and how we handle them.

## Reporting a Vulnerability

If you discover a security vulnerability, please **report it privately** to the project maintainers:

- Email: security@aceblocks.com (create an alias or use a monitored email)
- Subject line: "[AceBlocks Security]"

Please include:

- A description of the issue
- Steps to reproduce
- Screenshots or code examples if available
- Your contact information (optional)

**Do NOT create a public issue for security vulnerabilities.**

## Supported Versions

- The latest stable release of AceBlocks
- Any actively maintained versions (check the repository for release notes)

## Response Process

1. We will **acknowledge your report within 48 hours**.
2. We will investigate the issue and communicate updates via email.
3. Critical security issues will be **patched immediately** and released in the next update.
4. We may ask for your help testing the fix.
5. You may be credited in a public security acknowledgment if you choose.

## Reporting Guidelines

- Only report security issues. Non-security bugs should be submitted via the **GitHub Issues** page.
- Respect the privacy of other users and contributors.

## Security Updates

- All users are encouraged to update AceBlocks to the latest version as soon as security patches are released.
